# Start Bench Cut — UI

Angular + PrimeNG frontend for the **Start Bench Cut** game: drag three NBA players into **Start**, **Bench**, or **Cut** (roster-style "Fuck Marry Kill").

## Prerequisites

- **Node.js** 18.19.1+ (or 20.11.1+, 22+). Angular 18 and the build tools require Node 18+.
- **npm** 8+

If you're on Node 16, upgrade (e.g. from [nodejs.org](https://nodejs.org/) or via `nvm`).

## Run locally

```bash
npm install
npm start
```

Open **http://localhost:4200**. You should see three drop zones (Start / Bench / Cut), three draggable player cards, and Previous/Next to switch groups.

## What's in this repo

- **Angular 18** standalone app with **PrimeNG 18** (Aura theme).
- **Game screen**: three droppable zones (green / yellow / red), draggable player cards, PPG/RPG/APG stats, Previous/Next.
- **Data**: `NbaDataService` loads `data/nba-players-2025-26.json`; fallback to dummy data when the file is absent.
- **Python script**: `scripts/fetch_nba_players.py` — uses [nba_api](https://github.com/swar/nba_api) to fetch active 2025–26 players and stats, outputs JSON.
- **Models**: `src/app/game/models/player.model.ts` — `Player`, `RosterSlot`, `RosterAssignment`.

## Real player data (2025–26 season)

The app loads active NBA players and 2025–26 stats from JSON generated by the Python script (no API key).

1. Generate the file:
   ```bash
   pip install -r scripts/requirements.txt
   python scripts/fetch_nba_players.py
   ```
2. This writes `public/data/nba-players-2025-26.json`. Restart or refresh the app to see real players and PPG/RPG/APG.

If the file is missing, the app uses **dummy data**. Headshots are placeholders for now; you can add a headshot URL source later.

## Build

```bash
npm run build
```

Output is in `dist/start-bench-cut-plus-ui`.
